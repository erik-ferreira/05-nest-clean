# NestJS

- Framework muito mais opinado(traz muita opinião em como vai ser criada as coisas dentro dele) do que o express, koa e etc
- Tem 3 principais tipos de arquivos: controllers, services e modules
  - Exemplo: app.controller.ts, app.service.ts, e app.module.ts.

## Controllers

- Geralmente uma porta de entrada para a aplicação, geralmente via http
- Todo controller é identificado através de um decorator `@Controller()`
  - É possível adicionar um prefixo para as rotas do controller, ex: `@Controller('users')` e todas as rotas dentro desse controller vão ter o prefixo users, ex: `@Get()` vai ser acessada através de /users, `@Post()` vai ser acessada através de /users, etc

## Modules

- app.module.ts: é o arquivo principal da aplicação, onde são importados os outros módulos, controllers, services, conexão com banco de dados e etc;
- Esse arquivo nunca vai deixar de existir;
- Exemplo:

  ```typescript
  @Module({
    imports: [],

    // Quais controllers existem dentro daquele módulo
    controllers: [AppController],

    /*
    - Todas as dependências que os meus controllers podem ter, ou seja,
    dentro do AppController, é utilizado inversão de dependência
    para injetar o AppService, tendo assim que colocar nos providers
    
    - O nest vai automaticamente perceber que o meu controller(AppController)
    ta utilizando essa dependência, e vai fazer automaticamente a injeção de
    dependência, ou seja, vai criar uma instância do AppService e injetar
    dentro do AppController; Mas para a injeção acontecer, o meu service
    precisa estar com o decorator @Injectable()
    */
    providers: [AppService],
  })
  export class AppModule {} // Essa classe pode ter algo dentro dela, mas geralmente ela fica vazia, pois o que importa é o decorator @Module() que é onde são configurados os controllers, providers, imports e etc
  ```

### Decorators

- Geralmente são como funções que recebem um parâmetro e podem modificar internamente esse parâmetro, devolvendo ele modificado
